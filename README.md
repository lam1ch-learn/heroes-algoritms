# Heroes — Algorithms Project

## Реализованные алгоритмы

### 1. Генерация армии противника (`GeneratePresetImpl`)

**Алгоритм:** жадный (greedy).

Типы юнитов сортируются по эффективности:
- отношение атаки к стоимости;
- затем отношение здоровья к стоимости.

После сортировки юниты добавляются в армию до исчерпания лимита очков или
достижения ограничения в 11 юнитов одного типа.

**Алгоритмическая сложность:**
- сортировка типов юнитов: `O(t log t)`, где `t` — число типов (константа, `t = 4`);
- добавление юнитов: `O(m)`, где `m` — максимальное число юнитов в армии.

**Итоговая сложность:** `O(m)`.

---

### 2. Симуляция боя (`SimulateBattleImpl`)

**Алгоритм:** пошаговая симуляция боя по раундам.

На каждом раунде:
1. собираются все живые юниты обеих армий;
2. сортируются по убыванию силы атаки;
3. юниты ходят по очереди до завершения боя.

Юниты, погибшие до своего хода, пропускают его. Бой завершается, когда в одной
из армий не остаётся живых юнитов.

**Алгоритмическая сложность:**
- сбор очереди: `O(n)`;
- сортировка: `O(n log n)`;
- выполнение ходов: `O(n)`.

**Итоговая сложность:** `O(R · n log n)`, где `n` — число юнитов, `R` — число раундов.

---

### 3. Отбор доступных целей для атаки (`SuitableForAttackUnitsFinderImpl`)

**Алгоритм:** линейная фильтрация юнитов по рядам.

Юнит считается доступным для атаки, если:
- он жив;
- находится в первом ряду, либо
- не перекрыт юнитами предыдущего ряда по вертикали или диагонали.

**Алгоритмическая сложность:**
- линейный проход по юнитам противника.

**Итоговая сложность:** `O(n)`.

---

### 4. Поиск кратчайшего пути (`UnitTargetPathFinderImpl`)

**Алгоритм:** BFS (поиск в ширину) на двумерной сетке.

Поле представлено в виде сетки размером 27×21. Разрешено движение в 8 направлениях,
включая диагонали. Клетки, занятые живыми юнитами, считаются препятствиями.

Так как все перемещения имеют одинаковую стоимость, BFS гарантирует нахождение
кратчайшего пути.

**Алгоритмическая сложность:**
- в худшем случае посещаются все клетки поля.

**Итоговая сложность:** `O(WIDTH × HEIGHT)`.
